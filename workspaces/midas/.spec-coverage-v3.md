## FAIL Detail

### [SC-C1] CRITICAL: Frontend-Backend Band Threshold Mismatch

**Files:** `src/midas/regime/__init__.py` (backend) vs `apps/web/elements/pulse/PulseShell.tsx` (frontend)
**Spec:** `06-continuous-regime-rendering.md` S3; `09-surfaces-and-attention.md` S2

Backend thresholds (from `RegimeRenderer.get_band()`):

```python
if a_t >= 0.85: return CRISIS
if a_t >= 0.60: return URGENT
if a_t >= 0.30: return ELEVATED
return CALM
```

Frontend thresholds (from `deriveLayoutBand()`):

```typescript
if (a_t < 0.25) return "calm";
if (a_t < 0.5) return "elevated";
if (a_t < 0.75) return "urgent";
return "crisis";
```

**Impact:** For `a_t = 0.28`, the backend says CALM but the frontend renders ELEVATED. For `a_t = 0.55`, the backend says ELEVATED but the frontend renders URGENT. The compliance layer gates on one set of thresholds while the user sees a different visual regime. This violates spec 06 S3 and undermines the safety contract.
