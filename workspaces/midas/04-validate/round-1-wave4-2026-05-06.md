## Finding 1 (FIXED)

**Severity:** HIGH
**Category:** UI-Backend Defense / Spec Compliance
**Spec:** specs/09-surfaces-and-attention.md S3.3 + `rules/ui-backend-defense.md`

**Description:** Frontend `NotificationPreferences` slider `max=960` (16h) did not match backend's `[5, 120]` minutes validation range.

**Before:**
```tsx
min={60}   // 1h
max={960}  // 16h ← backend rejects >120
```

**After:**
```tsx
min={5}    // matches backend floor
max={120}  // 2h — matches backend ceiling exactly
step={5}
```

**Verification:** `grep -n "min=\|max=" NotificationPreferences.tsx` now returns `min=5, max=120`.
**Fix commit:** local edit.

**Why HIGH:** A user could slide to a value the backend rejects, getting a silent validation error without understanding why the save failed. Backend validation must match frontend affordance per `ui-backend-defense.md` Rule 1.
